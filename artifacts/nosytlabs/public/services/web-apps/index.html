Service · 01 — Web apps & sites

Small, sharp web work that ships.

Marketing sites, landing pages, focused web apps. The kind of project that should take a few weeks, ship clean, and keep working without a maintenance contract.

What you get

  • A real, deployed site — not a Figma file. Live URL, custom domain wired in, SSL valid, analytics installed under your account.
  • Source code in your GitHub org with an honest README — install, env vars, deploy command, and the gotchas we learned the hard way.
  • Performance that holds up — Lighthouse 90+ on mobile, Core Web Vitals in the green, gzipped JS under 150 kB on landing routes.
  • Accessibility baked in — semantic HTML, focus states, keyboard navigation, prefers-reduced-motion respected, WCAG AA contrast.
  • SEO that crawlers actually see — proper title/description, OG/Twitter cards, JSON-LD schema, sitemap.xml, robots.txt, llms.txt.
  • One round of post-launch tweaks within two weeks of handover — typo fixes, copy tweaks, the small things you only spot once it’s live.

How this one got built

The site you’re reading right now is the reference. Vite + React + TypeScript, deployed static. Static cosmos hero (76 KB WebP) served at every viewport — no autoplay video, no CDN tax on mobile or low-bandwidth visitors. Static SEO skeleton inside the React root so non-JS crawlers see real content. CSP, Permissions-Policy, and structured data all hand-written. Source is in the open — see how it’s wired together before you commit.

How it works

  1. Email a one-paragraph brief. What you’re shipping, who it’s for, when it has to be live.
  2. Reply within 2 business days — fit/no-fit, scope questions, draft estimate.
  3. Fixed-scope written quote — deliverables, timeline, what would change the price.
  4. Build in the open by default; private repo when the work is yours.
  5. Code transferred to your org on completion. You own the build, the deploy, and the next decision.

Honest answers

What does it cost?

Scope-based, written quotes. A focused marketing site with 3–5 sections lands in the low four figures; a small custom web app with auth, a database, and a couple of integrations is meaningfully more. We don’t publish a price list because every project has different complexity — but we’ll send you a real number within two business days of hearing the brief, and we won’t move the number once we agree.

How fast can it ship?

Two to six weeks for most projects. If you need it in five days, you need a different partner — focused work doesn’t survive a sprint to nowhere, and rushing is how bugs ship.

Will I own the code?

Yes. Repository transferred to your GitHub org on completion, with deploy keys / secrets handed over. No vendor lock-in, no “managed service” contract you have to keep paying. If you want the studio to keep maintaining it, that’s a separate conversation.

What about ongoing maintenance?

The studio keeps the door open for retainer-style support after launch — security updates, dependency bumps, occasional tweaks — but it’s optional. The code is yours and any competent React developer can take over.
